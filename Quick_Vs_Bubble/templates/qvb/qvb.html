QuickSort versus BubbleSort!

I thought it would be nice to see HOW MUCH faster QuickSort is than BubbleSort.

With the additional comparison between blocking and asynchronous QuickSort.

On the one hand I thought it was a bit unfair to compare BubbleSort with an asynchronous QuickSort, as BubbleSort cannot be run in an asynchronous manner.

On the otherhand its BubbleSorts fault to not be asynchronous-compatible..

In the end I thought: Why not both?

To use an array of custom size, use the get parameter "size".

Default size is 100. The array will be filled with values between [0, size). (An array of size 1000 gives an impressive comparison - but even then BubbleSort takes forever. For more than 5000 barely anything is visible.

Example:

andreasecke.de/probe/qvb?size=1000

Warning: Only the first parameter will be used. Any other parameter will be discarded!

To use a custom array, use the get parameter "arr" and split the numbers with commas.

The default array contains 100 randomized values between [0, 100).

The sorted/final Array (sorted independently of the visual ongoings), will be logged into the console.

Example:

andreasecke.de/probe/qvb?arr=5,2,5,6,8,4,2

Warning: Only the first parameter will be used. Any other parameter will be discarded!

The "Sleep" duration after every comparison is 50ms.
